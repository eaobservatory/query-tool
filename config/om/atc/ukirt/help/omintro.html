Introduction to using ORAC for Observing

Follow the links or the index to find detailed help about each stage of the ORAC-OM. The following is a quick over-view of how it is used, and there is a one page "crib sheet" here.

The ORAC observing system is designed to use Science Programs which have been pre-prepared using the ORAC-OT, and submitted to the UKIRT database. These Science Programs contain all the information needed to obtain your data - such as instrument configuration and co-ordinates.

Normally you will have created one Science Program for your PATT run, which contains many observations (one or more for each target). You can add definitions for new observations during observing by using the OT at the summit and submitting them to the database.

The ORAC-OM controls the login procedure and ensures that only you (and your collegues) have access to your program and the co-ordinates and observing plans therein. Once you have gone through the startup procedure, it will display a listing of programs you have stored in the database. (You may have programs for more than one PATT run for example). You chose the one for the current observing run and then the OM will display all the Observations you have defined for your program. When you select which observation you want to carry out, the selected observation is passed to the Observation Sequencer for execution. The observation sequencer has a console which allows you to stop and start the sequence, take "movies" to check out the instrument configuration, etc. When you have finished a given Observation you simply choose the next one from the list on the ORAC-OM window.

To Start-up ORAC for observing, log into the observing control workstation and type the command orac_om to start up the ORAC Observing Manager (ORAC-OM). Here is a link to the step by step guide to running using the ORAC Observing Manager which can also be found from the help file index.

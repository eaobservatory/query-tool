QT Help Desk

The OMP Query Tool (QT) is provided to allow observers at the telescope (and possibly support scientists in Hilo) to query the MSB and Project databases to determine the most appropriate observations to take given the current weather and scheduling conditions.

The QT can be thought of as a short-term scheduler. It can be used to determine the best observation to take now and to allow the observer to plan a single nights observing (assuming estimates of weather conditions and knowledge of current instrument availability) but it will not be able to estimate the mid- to long term schedule for the telescope.

In addition to providing a means to query the databases the tool will also be used to provide a means to store calibrations that can not yet be observed.

Note that only ONE query tool can be "live"  for any telescope at any time.  If you attempt to execute a Query Tool while one is already running, it will be brought up in scenario mode.  This will let you do most things except sending items for execution.  Locking is done via a lockfile, the location of which will be displayed when the tool starts and a lock is present.

The Information contained in this help file is designed to expalin each of the features of the QT.

The Query Screen

When the QT is first started the user is presented with several panels.  This section will descibe each of those panels and guide the user in their use.  The actual layout of the screen will depend on whether the user is running hte QT for the JCMT or UKIRT (or any other telescope) since the interface is built dynamically.  However, some panels are common to both.  These are the Data Panel, Search Paramters Panel, Atmosphere Panel, Instrument Panel and Results Panel.  This screen is designed to allow users to query for suitable projects (either a specific project or ones meeting the query contraints).  The user should be aware that the QT is dumb in that it peforms minimal error checking and relys on other systems to validate most of the query (the exception being the date search parameters).   Each of the main components of the QT Search Window will now be discussed in more detail.

The Data Panel 


The Data Panel shows current information which could affect observations, as well as containing some of the command buttons.  The two time panels at the bottom of the panel show UTC and local time respectively, and a satellite image of the local region is also shown to allow users to assess upcoming weather conditions.  The "Current Info" shows current conditions if they are being monitored.  In addition, there are three command buttons.  Each item is described in more detail below.

Search Button

Once a user has selected their search criteria on the Search Parameters Panel,  pressing this will execute the search and return the set of results matching the criteria.  These results will appear as a selectable list in the Results Panel.

Fetch MSB Button

Once a set of results is obtained and one selected from the list in the Results Panel, this button will show more detail of the selected MSB.  This will appear as a tabbed pane, so the user can easily go back to the results and select a new MSB without performing the query again.

Exit Button

Current Info

Displays current atmospheric and telescope information providing the appropriate monitoring tools are executing.  The current set of displayed information is:

  1. 225 GHz Tau - The optical depth of the atmosphere at 225 GHz measured either by the Caltech Submillimeter Observatory at Mauna Kea or by the JCMT's Water Vapour Meter.
  2. Seeing - currently not measured
  3. Airmass - The normalised thickness of the atmosphere ranging from 1 at the zenith to infinity at the horizon.  Related to the elevation of the telecope by the fomula airmass = cosec (90-elevation)

If these parameters are not measured then the values will appear as "-----".

The "Set Current" button will set put the current values into the search criteria is applicable (see Atmospher Panel).  In addition, it will set the current brightness of the Moon in the Moon Panel (again, if applicable),  and reset the Observation Date and Time on the Search Parameters Panel to the current if they were prevously changed.

Satellite Image

In the JAC version of the tool, this shows an GOES-10 IR Image of the Hawaii Region from the Mauna Kea Weather Center (original data from NWS).  This image shows the brightness temperature at 10.2 to 11.2 microns; coldest cloud tops showing white.

Other images can be shown by altering the value of satellitePage entry in the file qtSystem.conf

Search Parameters Panel


The Search paramters panel allows users to define criteria on which to search the project database.  The user can supply as much or as little information as they need.   All fields are optional - no values need to be specified.

PI or CoI
 The surname of the Principle Investigator or Co-Investigator .  If left blank, this is not used in searching.
Project ID
 A unique Project ID.  If left blank, this is not used in searching
Semester
 The semester for the project(s) to be returned.  If left blank, the current semester is assumed.
Hour Angle
 Limit the results returned to those in the specified range of hour angles.  If left blank, this is not used in the search.
Airmass1
 Limit results to those projects requiring airmass between specified values.  The definition of minimum and maximum is umimportant as it will be interpreted before the query (e.g. a mimmum value of 1.2 and maximum of 2 wll be identical to a search with minimum of 2 and maximum of 1.2).  Allowable range is 1.0 to 3.0.  If left blank, this is not used in searching.
If the user uses the "Set Current" facility on the Data Panel, this will set the airmass range to +/- 20% of the current airmass.
Elevation1 - JCMT Only
 Limit results to those projects requiring an elevation between specified values.  The definition of minimum and maximum is umimportant as it will be interpreted before the query (e.g. a mimmum value of 30 and maximum of 45 wll be identical to a search with minimum of 45 and maximum of 30).  Allowable range is 90.0 to 0.0.  If left blank, this is not used in searching.
Observation Date
 Use this to "look ahead" to see what will be observable in the future.  If the time (seconds field) is incrementing, the current date and time is assumed.  Editing either field will stop the time incrementing, and allow the user to type in the required information in Local Time.  The Date and time  fields must be entered in the form yyyy-mm-dd and  hh:mm:ss.  Any error in this format will mean the current time is assumed.
Also note that if the user enters a valid date for observation, the moon tag (if available) will not be searched.
Duration
 Limits the results to those prohjects which can be performed within the specified duration.  Typically used towards the end of a shift to see if any short projects can be completed.  If left blank, then project duration is not used in searching.
Clouds - UKIRT only
 Allows users to select the current level of cloud coverage from a pre-defined selection:
  • Photometric - no cloud clover
  • Thin - High level thin cirrus 
  • Thick - High level thich cirrus - only bright sources can be oberved
If not selected, not used in searching.
Country - JCMT only
 Allows users to select the country owning the project.  If not selected, not used in searching.
Atmosphere
 Allows users to set a tau and seeing value.  Any projects which can be performed at the specified tau and/or seeing will be returned from the search.  If left blank, these will not be used for searching.
If the user uses the "Set Current" facility on the Data Panel, this will set the current tau and seeing (if available) in these fields.
Moon - UKIRT only
 Allows users to set the current brightness of the moon.  When the QT starts up it is set to the current brightness.  Using the "Set Current" button updates this field.  The user can override the entry.

1: Note that if both Airmass and Elevation are used for searching, the results will be returned from the interscetion of these ranges.

Staging Area


The Staging Area shows more details of a selected MSB.  The Staging Area consists of the following Panels:

  • The Data Panel - as described previously.  Note the from the Staging Area, the "Search" and "Fetch MSB" buttons have no meaning and should not be used
  • the Program List - showing the current Observations to be perfomed in the MSB
  • The Deferred List - showing deferred observations from this and previous MSBs
  • The Observer Notes - showing notes for observing this MSB
  • Calibration Menu
  • Trash Can/Execution Panel
The purpose of this screen is to allow users to select specific observations for observation, or defer observations.  It allows a fine level of control so that it is not necessary to perform the observations in the specified order, but the observations can be optimised to allow most efficient use to telescope time.

Each of these will now be looked at separately in the following sections.

The Program List

The Program List shows all of the title of the observations in the selected science program.  There may be only a single or many observations.  In the case where there are more observations than will fit on the screen, the panel will add a scrollbar to allow the user to view all of the observations.

Observations in the Program List will occur in one of two colours:

  • Blue indicating that an observation is optional
  • Black indicating that the observation is mandatory
The difference between the two is fairly simple; optional observations can be deleted (by dragging them to the Trash Can) or deferred (by dragging them to the Deferred List).  Typically this will only be done for calibration observations.  Mandatory observations are typically observations on a science target, and can not be deleted or deferred.  Any observation which is deleted or deferred is only done so 'notionally'; that is to say is the MSB is retrieved again, the observations will reappear.  So if the user accidentally deletes an observation is can be retrieved by fetching the MSB again from the Query Screen.

A number of other functions are available from this panel.  
  • Double clicking on any observation will send that observation for execution.  
  • Right clicking on a selected observation will allow the user to edit some of the observational parameters (currently this is limited to the exposure time).  
Like deferred and deleted observations, the edits are only valid to the current session.  Fetching the MSB again from the Query Screen will reset the values to their originals.  Both optional and mandatory observations can be edited or sent for execution.

Deferred List


The Deferred List shows the list of deferred observations.  When the QT starts up, the user is asked whether they want to use the currrent set of deferred observations if any exist.  Selecting "No"  to this will delete all of the existing deferred observations from the system,  otherwise the existing observations will appear in this list.  Similary, on Exit, the user is asked whether they want to save any of the deferred observations which exist. Selecting "No" will delete any observations deferred during this and/or previous sessions, while "Yes" will preserve them for use in the next session.

Observations will appear in one of two colours:

  • Blue indicating that the observation has been deferred but not observed
  • Red indicating that the observation has been observed
In addition, once a deferred observation has been executed, it will be marked as done and time-stamped with the time at which the observation was executed (in local time).  This allows the user to redo an observation at a later time for another project or use the same observation again without the need to repeat it for each MSB.

The Deferred List can be populated by either dragging optional observations from the Program List to the Deferred List, or selecting a standard calibration obervation from the Calibrations Menu.  If two identical observations are entered (for istance the MSB has two identical calibrations - one before and one after a science observation), it will appear only once on the deferred list.

Deferred Observation can only be executed by using the "Send for Execution" button and are not editable is the way observations in the Program List are editable.  However, if the Observation is first edited in the Program List and then copied to the deferred list, the edited values will be maintained.  Items can be deleted from this list by dragging the to the Trash Can.

Calibrations Menu

The calibrations menu on the main menu maenu bar is active when the Staging Area is active.  It allows users to select sets of standard calibration procedures for the telescope to be performed.  On selecting an item from the menu, the entry is added to the deferred list and can either be executed immediatley or at a suitable time in the future.

Observer Notes

This panel is for notes to aid the observer in performing the MSB.  This panel is not editable.  It could be used, for example, to specify to an oberver that the obervation times on specific items should not be edited.  Each MSB may only have, at most, one observer note.

Do NOT try to edit this field, on pain of certain death!

Sending for Execution

Once you have selected an observation from either the Program or Deferred List, use the "Send for Execution" button to send it to the sequence console.

Do not try to retrieve deleted items from the trash can.  Due to the efficient trash collection service in Hawaii, items are removed as soon as they are dumped!
